下面给你一个**“第一阶段：12×512 → 49,412 维 multi-hot 概率头 + 概率软融合 CLIP embedding”**的可落地专业方案。目标是：在不把信息挤压到单一 token 的前提下，把视频的视觉证据转成一个**稀疏语义分布**，并且把该分布映射回 CLIP 文本空间用于后续阶段（短语生成/句子生成）。

---

## 0) 符号与张量形状

* 词表大小：(V=49{,}412)（CLIP 的 BPE vocab）
* 帧数：(T=12)
* 维度：(D=512)
* batch：(B)

输入（你已提取好的 CLS）：
[
\mathbf{X}\in \mathbb{R}^{B\times T\times D}
]
其中 (\mathbf{X}_{b,t}) 是第 (b) 个视频第 (t) 帧的 CLIP ViT-B/32 最终层 CLS（img token）。

CLIP 文本 embedding 矩阵（预先离线取出并缓存，建议冻结）：
[
\mathbf{E}\in \mathbb{R}^{V\times D}
]
(\mathbf{E}_i) 为第 (i) 个 vocab token 的文本 embedding（建议做 L2 normalize）。

输出：

* 词汇 logits：(\mathbf{L}\in\mathbb{R}^{B\times V})
* 概率：(\mathbf{P}=\sigma(\mathbf{L})\in\mathbb{R}^{B\times V})

软融合后的视频语义表示（两种常用形式）：

1. 单向量融合（给后续句子级聚合也很方便）
   [
   \mathbf{g}\in\mathbb{R}^{B\times D}
   ]
2. 多 token 融合（避免挤压，给第二阶段短语生成更合适）
   [
   \mathbf{S}\in\mathbb{R}^{B\times M\times D},\quad M\ll V
   ]

---

## 1) Block 设计（推荐：轻量时序编码 + CLIP 词表权重绑定头）

### 1.1 输入归一化与时间位置编码

CLIP 空间的稳定性很重要，推荐先做 LayerNorm + 可选 L2 normalize：

[
\tilde{\mathbf{X}}=\mathrm{LN}(\mathbf{X}),\quad
\hat{\mathbf{X}}*{b,t}=\frac{\tilde{\mathbf{X}}*{b,t}}{|\tilde{\mathbf{X}}_{b,t}|_2+\epsilon}
]

加入可学习时间位置编码 (\mathbf{P}\in\mathbb{R}^{T\times D})：
[
\mathbf{H}^{(0)}*{b,t}=\hat{\mathbf{X}}*{b,t}+\mathbf{P}_t
]

> 若你有“k 段抽帧”概念，也可以加 segment embedding (\mathbf{S}_{seg})；但你当前固定 12 帧，先不加也可。

---

### 1.2 时序聚合器：小型 Transformer Encoder（2–4 层足够）

采用 Pre-LN 结构的 encoder layer（推荐 2 层起步，便于先把第一阶段跑稳）：

对第 (l) 层：
[
\mathbf{U}=\mathbf{H}^{(l-1)}+\mathrm{MHSA}(\mathrm{LN}(\mathbf{H}^{(l-1)}))
]
[
\mathbf{H}^{(l)}=\mathbf{U}+\mathrm{FFN}(\mathrm{LN}(\mathbf{U}))
]

建议超参（起步配置）：

* 层数 (L=2)（不够再加到 4）
* heads (h=8)（每头 64）
* FFN hidden (=2048)
* dropout=0.1（含 attention dropout）
* 可选 stochastic depth=0.05（如果 L≥4）

输出：
[
\mathbf{H}=\mathbf{H}^{(L)}\in\mathbb{R}^{B\times T\times D}
]

---

### 1.3 视频级向量池化：推荐 [VID] learnable token

加入一个可学习聚合 token (\mathbf{z}\in\mathbb{R}^{1\times D})，拼到序列前：
[
\mathbf{H}'=\mathrm{concat}(\mathbf{z},\mathbf{H})\in\mathbb{R}^{B\times (T+1)\times D}
]
再经过同样的 encoder（你也可以把 [VID] 直接放进上面的 encoder 输入）后取第 0 个位置：
[
\mathbf{v}=\mathbf{H}'_{:,0,:}\in\mathbb{R}^{B\times D}
]

> 相比 mean pooling，[VID] 在多语义聚合时更稳，且便于后续加门控/对齐。

---

### 1.4 词汇概率头：与 CLIP 文本 embedding 权重绑定（关键点）

你要的 vocab 输出很大，最专业/最省参数的方式就是**用 (\mathbf{E}) 做分类器权重**（weight tying），并用 cosine/temperature 做稳定对齐。

先做一个小投影（允许模型在 CLIP 空间内做轻微旋转/缩放）：
[
\mathbf{u}=\mathrm{Normalize}(\mathbf{W}_p\mathbf{v}+\mathbf{b}_p)\in\mathbb{R}^{B\times D}
]
其中 (\mathbf{W}_p\in\mathbb{R}^{D\times D})（可用低秩 LoRA/Adapter 以减少参数，后面给选项）。

对 (\mathbf{E}) 做 normalize：
[
\hat{\mathbf{E}}_i=\frac{\mathbf{E}_i}{|\mathbf{E}_i|_2+\epsilon}
]

logits 用 cosine similarity + learnable temperature：
[
\mathbf{L} = \tau \cdot \mathbf{u}\hat{\mathbf{E}}^\top + \mathbf{b}
]

* (\tau) 是可学习标量（初始化 10 或 14 附近都可以）
* (\mathbf{b}\in\mathbb{R}^{V}) 为可学习 bias（可选但我建议加，能显著缓解长尾先验偏置）

得到概率：
[
\mathbf{P}=\sigma(\mathbf{L})\in\mathbb{R}^{B\times V}
]

**形状检查**

* (\mathbf{u}:\ (B,512))
* (\hat{\mathbf{E}}^\top:\ (512,V))
* (\mathbf{L}:\ (B,V))

---

## 2) 概率软融合 CLIP embedding（给第二/三阶段的“多关键 token”表示）

### 2.1 Top-M 稀疏语义 token（推荐，避免密集 49k 混合）

取每个样本的 top-M 概率索引：
[
\mathcal{I}*b=\mathrm{TopM}(\mathbf{P}*b),\quad |\mathcal{I}*b|=M
]
取对应 embedding 并做权重门控：
[
\mathbf{S}*{b,j} = \alpha*{b,j}\cdot \hat{\mathbf{E}}*{\mathcal{I}_b(j)}
]
其中权重可选：

* 直接用 (\alpha_{b,j}=\mathbf{P}_{b,\mathcal{I}_b(j)})
* 或做局部归一：(\alpha=\mathrm{softmax}(\mathbf{L}_{topM}))（更像 mixture）

得到：
[
\mathbf{S}\in\mathbb{R}^{B\times M\times D}
]

建议 (M=32) 或 64 起步（第二阶段短语集合学习更舒服）。

---

### 2.2 单向量融合（可选，给第三阶段做全局条件）

[
\mathbf{g}=\mathbf{P}\hat{\mathbf{E}}\in\mathbb{R}^{B\times D}
]
但注意这是密集混合，训练早期可能被大量负类噪声污染；如果用它，建议只用 top-M 后再加权求和。

---

## 3) 参数学习策略（冻结/微调的推荐组合）

### 3.1 冻结项（建议先冻结，先把第一阶段训稳）

* 冻结 CLIP image backbone（你当前已离线提 CLS，本质就是冻结）
* 冻结 CLIP text embedding (\mathbf{E})（强烈建议）

### 3.2 学习项（第一阶段需要学的）

* 时间位置编码 (\mathbf{P})
* Transformer encoder 参数（2–4 层，较少）
* [VID] token (\mathbf{z})
* 投影层 (\mathbf{W}_p,\mathbf{b}_p)
* 温度 (\tau)
* bias (\mathbf{b}\in\mathbb{R}^{V})

> 若显存/参数敏感：(\mathbf{W}_p) 可改为 **低秩**：(\mathbf{W}_p=\mathbf{I}+ \mathbf{A}\mathbf{B})（LoRA），rank=8/16 即可。

---

## 4) 训练目标：多标签 BCE +（强推荐）长尾与负类抑制

### 4.1 你的标签平滑（multi-label 专用，不建议“负类拉到 0.1”那种）

你标签是 multi-hot：(\mathbf{y}\in{0,1}^{B\times V})。

对 multi-label 来说，更稳的平滑方式是**正类往 1-ε 收缩，负类给极小 (\epsilon_0)**（否则 49k 负类会引入巨量噪声梯度）：
[
\tilde{y}*{i}=
\begin{cases}
1-\varepsilon*+,& y_i=1\
\varepsilon_-,& y_i=0
\end{cases}
]
推荐起步：

* (\varepsilon_+=0.05\sim 0.1)
* (\varepsilon_-=10^{-5}\sim 10^{-4})（或 (\varepsilon_-=\frac{\varepsilon_+}{V-1}) 量级）

### 4.2 损失函数：Asymmetric Loss（强烈推荐给 49k 长尾）

标准 BCE 在超多负类下会被负类主导。Asymmetric Loss（或 Focal BCE 的不对称版本）更适合：

设 (p=\sigma(l))。
[
\mathcal{L}=
-\sum_i \left[
\tilde{y}*i (1-p_i)^{\gamma*+}\log(p_i)
+
(1-\tilde{y}*i), p_i^{\gamma*-}\log(1-p_i)
\right]
]

推荐起步：

* (\gamma_+=0)（正类不压）
* (\gamma_-=4)（强压负类）
* 可加 negative clipping：(p_i\leftarrow \max(p_i, c)) 对负类（例如 (c=0.05)）进一步稳定早期训练

如果你坚持用 BCEWithLogits，也至少加：

* `pos_weight`（按类别频率或全局比例）
* 或做 logit adjustment（按类先验 ( \pi_i ) 加偏置：(l_i \leftarrow l_i + \log \pi_i)）

---

## 5) 计算优化（很关键）：全量 49k vs 负采样训练

### 5.1 全量训练（实现最简单）

直接算 ((B,V)) logits 与 loss。
在 batch 不大时是可行的（显存主要来自 logits）。

适用：

* B ≤ 64（看你 GPU）
* 正样本密度不算极稀（否则浪费算力）

### 5.2 推荐：正类全保留 + 负采样（训练更快、更稳）

每个样本：

* 收集正类集合 (\mathcal{P}_b)
* 从负类里采样 (K) 个 token：(\mathcal{N}_b)，(K=2048\sim 8192)
* 只对索引集合 (\mathcal{S}_b=\mathcal{P}_b\cup \mathcal{N}_b) 计算 logits 和 loss（embedding gather）

训练仍然学到同样的 (\tau,\mathbf{W}_p)，推理时再全量算 ((B,V))。

---

## 6) 优化器、学习率、训练细节（可直接照抄起跑）

### 6.1 优化器

**AdamW**：

* betas = (0.9, 0.98) 或 (0.9, 0.999)
* weight_decay = 0.05（Transformer 类参数）
* 不对 LayerNorm/bias 做 weight decay

### 6.2 学习率（分组，稳定）

建议 parameter groups：

1. Temporal encoder + pos emb + [VID]：lr = (1e{-4})
2. 投影 (\mathbf{W}_p) + 温度 (\tau) + vocab bias (\mathbf{b})：lr = (2e{-4})（头稍快）
3. 若你后续想微调 CLIP（此阶段先不建议）：lr = (1e{-5}) 并强正则

### 6.3 调度器

* warmup 5% steps（线性 warmup）
* 之后 cosine decay 到 1e-6

### 6.4 稳定性技巧

* mixed precision（fp16/bf16）
* gradient clipping：1.0
* dropout 0.1
* early stopping 监控 mAP / micro-F1 / recall@K（K=64/128）
* 阈值策略：推理时可用 per-class 阈值（基于先验频率）或固定阈值 + top-K

---

## 7) 你最终拿到的第一阶段“可交付接口”（供第二/三阶段用）

给后续阶段建议输出以下三样（都能从上面直接得到）：

1. **词汇概率分布**
   [
   \mathbf{P}\in\mathbb{R}^{B\times V}
   ]

2. **Top-M 语义 token 序列**（你要的“多个关键有效信息 Token”）
   [
   \mathbf{S}\in\mathbb{R}^{B\times M\times 512}
   ]

3. **全局视频向量**（可选，给第三阶段条件/价值网络更好用）
   [
   \mathbf{v}\in\mathbb{R}^{B\times 512}
   ]

---

## 8) 两个常见坑（提前规避）

1. **负类平滑过大**：如果你把负类从 0 平滑到 0.05/0.1，在 49k 维下会制造巨量虚假梯度，训练会非常难收敛。务必用我上面那种 (\varepsilon_-) 极小的方式。
2. **不做长尾处理**：只用纯 BCE 往往会学成“全 0”或“只会高频 token”。至少用 Asymmetric Loss / focal / pos_weight 其一。
